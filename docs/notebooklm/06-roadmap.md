## 二、未來發展方向

### 方向 1：進階 RAG — 讓回答更精準

**現狀**：基本 RAG pipeline 已運作良好，但有優化空間。

**規劃功能**：

| 功能 | 說明 | 效果 |
|------|------|------|
| **Hybrid Search** | 結合向量語意搜尋 + 關鍵字搜尋 | 解決純語意搜尋漏掉精確匹配的問題（如產品編號、專有名詞） |
| **Query Rewriting** | AI 自動改寫使用者問題再搜尋 | 模糊提問也能找到對的資料 |
| **Retrieval Grading** | AI 評估檢索結果是否真的相關 | 過濾掉低品質的檢索結果，減少「答非所問」 |
| **Adaptive Retrieval** | 根據問題類型動態調整搜尋策略 | 簡單問題少搜、複雜問題多搜，平衡品質和速度 |

**對客戶的價值**：回答準確率進一步提升，減少需要轉接真人的比例。

### 方向 2：更多通訊管道 — 觸及更多客戶

**現狀**：已支援 Web Chat 和 LINE Bot。

**規劃管道**：

| 管道 | 適用場景 | 技術難度 |
|------|---------|---------|
| **Facebook Messenger** | 有 FB 粉絲頁的商家 | 中（Webhook 模式，類似 LINE） |
| **Instagram DM** | 年輕客群、視覺導向品牌 | 中（透過 Meta Graph API） |
| **WhatsApp Business** | 國際客戶 | 中（透過 WhatsApp Cloud API） |

**架構優勢**：系統已有 Channel Router 設計（Web + LINE），新增管道只需加一個 Webhook 入口，核心 Agent 邏輯完全不用改。

**預估工時**：每個新管道約 1-2 週。

### 方向 3：無限擴展的業務工具 — 客戶想做什麼就做什麼

**核心概念**：只要是 Web 上能做的事，做成 MCP Tool 就能讓 AI Agent 幫客戶做。

**已實現範例**（窩廚房）：

| Tool | AI 能做的事 |
|------|-----------|
| `query_products` | 查商品、比價格、篩分類 |
| `query_courses` | 查課程空位、查講師、查日期 |

**未來可擴展的方向**：

| 業務場景 | MCP Tool | AI 能做的事 |
|---------|---------|-----------|
| 訂單管理 | `query_orders` | 幫客戶查訂單狀態、物流進度 |
| 預約報名 | `create_booking` | 幫客戶直接報名課程、預約服務 |
| 退貨處理 | `create_return` | 引導客戶填退貨單、查退款進度 |
| 會員查詢 | `query_member` | 查詢點數、優惠券、會員等級 |
| 庫存查詢 | `query_inventory` | 即時回覆商品是否有貨 |
| 轉接真人 | `transfer_agent` | 複雜問題自動轉接真人客服 |

**MCP 是 AI 生態系的趨勢**：

MCP（Model Context Protocol）是由 Anthropic 提出的開放標準，定義了 AI Agent 如何與外部工具溝通。目前已被 OpenAI、Google、Microsoft 等主要 AI 廠商採納，成為 AI 工具串接的業界標準。

這代表：
- **生態系持續擴大** — 越來越多第三方服務會提供 MCP 接口，未來可直接串接而不需自行開發
- **不會被淘汰** — 我們選擇的技術標準是業界主流，不是自創的封閉協議
- **開發成本持續降低** — 隨著 MCP 生態成熟，現成的 Tool 越來越多，客製化開發量遞減

**對部長的意義**：

- **不需要重新開發系統** — 新增業務功能只需要加一個 MCP Tool
- **客戶需求彈性大** — 不同產業、不同客戶，各自需要的 Tool 不同，按需開發
- **每個 Tool 都是獨立的** — 加一個不會影響其他功能，風險低
- **這是 LumineAI 做不到的** — LumineAI 只有 Web 搜尋和圖片生成，無法串接客戶自有系統
- **符合業界標準** — MCP 已被主要 AI 廠商採納，我們的架構跟著業界走

## 四、LLM 模型降價趨勢 — 成本只會更低

### 歷史降價幅度

LLM API 價格在過去兩年大幅下降：

| 時間 | 事件 | 影響 |
|------|------|------|
| 2024 → 2025 | GPT-4o 取代 GPT-4 | 同等品質，價格降 75% |
| 2025 → 2026 | GPT-5 系列 + DeepSeek V3 | 整體市場價格再降 50-80% |
| 持續 | 開源模型崛起（Llama、Mistral） | 長期價格壓力持續 |

### 對我們的意義

- **自建系統直接受益** — 模型降價，我們的營運成本自動降低
- **LumineAI 不會降價** — 平台方吃了中間利潤，不會把降價紅利傳給客戶
- **今天的投資，明天更划算** — 基礎設施成本固定，LLM 成本持續下降，毛利率只會越來越高

### 未來成本預估

| 時間 | 預期 LLM 價格變化 | 自建每次對話成本 |
|------|-----------------|---------------|
| 2026 現在 | 基準 | 2.3-2.6 TWD |
| 2026 下半年 | 預估再降 30-50% | 1.2-1.8 TWD |
| 2027 | 預估再降 50% | < 1 TWD |

> 當每次對話成本低於 1 TWD 時，AI 客服的邊際成本趨近於零 — 這是自建系統的長期優勢。
