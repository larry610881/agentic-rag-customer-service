# 定價策略與成本營收預算

## 一、成本結構拆解

自建方案的成本結構完全透明，由三大塊組成：

| 成本類型 | 說明 | 特性 |
|---------|------|------|
| **基礎設施（固定成本）** | GCP 雲端服務：運算、資料庫、快取 | 不隨對話量變動，隨租戶數階梯升級 |
| **LLM API（變動成本）** | 呼叫 AI 模型的費用 | 按用量計費，用多少算多少 |
| **向量資料庫（隨規模變動）** | 初期含在 Cloud SQL，規模化後獨立 | < 30 家客戶零額外成本 |

傳統方案把這三者打包成一個不透明的 token 包（如 18M tokens / 50,000 TWD），無法拆開看成本結構，也無法享受 LLM 降價紅利。

## 二、單租戶成本明細

### GCP 基礎設施

| 服務 | GCP 產品 | 規格 | 月費估算 | 備註 |
|------|---------|------|---------|------|
| Backend API | Cloud Run | 1 vCPU, 1 GiB, min: 0 | ~$0-15 | 按用量計費，idle 趨近 $0 |
| PostgreSQL + pgvector | Cloud SQL | 1 vCPU, 3.75GB RAM, 20GB SSD | ~$50 | 含自動備份 + 向量搜尋 |
| Redis | Memorystore | 1GB Basic | ~$35 | Session / cache / rate limit |
| Frontend | Firebase Hosting | 靜態檔案 + CDN | ~$0-5 | 低流量幾乎免費 |
| 其他 | VPC + Registry + Monitoring | — | ~$8 | 必要基礎元件 |
| **基礎設施小計** | | | **~$93-113/月** | |

### LLM API 三方案

> 假設單租戶低用量：每日 10 次對話，每對話 5 輪。

| 方案 | LLM 選擇 | 月 API 費用 | 適用場景 |
|------|---------|-----------|---------|
| **Budget** | GPT-5 nano + Gemini Flash-Lite | **~$0.14** | 簡單 FAQ、預算極度有限 |
| **Recommended** | GPT-5 mini + GPT-5 nano（路由） | **~$0.49** | 一般企業客服，品質與成本平衡 |
| **Premium** | Claude Sonnet 4.6 + GPT-5 nano（路由） | **~$4.09** | 高品質需求、複雜推理 |

### 向量資料庫策略

**初期選擇 Cloud SQL pgvector — 零額外成本**：

- 向量搜尋直接跑在 $50/月的 Cloud SQL instance 上，不需額外服務
- 同一個 DB instance 存 business data + vector data，減少基礎設施成本
- 單租戶向量量預估 < 500K，pgvector HNSW 效能綽綽有餘（P99 ~25ms）
- GCP 託管服務：自動備份、自動升級、IAM 整合

### 單租戶月費小計

| 方案 | 基礎設施 | API 費用 | 月總費用 |
|------|---------|---------|---------|
| Budget | ~$93 | ~$0.14 | **~$93** |
| Recommended | ~$93 | ~$0.49 | **~$93** |
| Premium | ~$93 | ~$4.09 | **~$97** |

> **關鍵洞察**：基礎設施是固定成本（占 95%+），LLM API 按用量計費。規模化後，基礎設施共攤，邊際成本趨近 API 費用。

## 三、規模化成本 + 向量 DB 轉換邊界

### 向量數量成長預估

| 客戶數 | 每戶文件 chunks | 總向量數 | 向量 DB 建議 |
|--------|----------------|---------|-------------|
| 1-5 家 | ~50K | < 250K | Cloud SQL pgvector（同 instance） |
| 5-10 家 | ~50K | 250K-500K | Cloud SQL pgvector（綽綽有餘） |
| 10-30 家 | ~50K | 500K-1.5M | Cloud SQL pgvector（仍可行，需調優） |
| 30-100 家 | ~50K | 1.5M-5M | **⚠️ 轉折點：評估遷移 Qdrant** |
| 100-1000 家 | ~50K | 5M-50M | **Qdrant 自架**（分散式，水平擴展） |

### 效能對比（Benchmark 參考）

> 測試條件：1536 維向量、1 vCPU 4GB RAM

| 資料量 | pgvector QPS | pgvector P99 | Qdrant QPS | Qdrant P99 |
|--------|-------------|-------------|-----------|-----------|
| 100K | ~200 | ~10ms | ~800 | ~3ms |
| 500K | ~100 | ~25ms | ~500 | ~8ms |
| 1M | ~60 | ~45ms | ~300 | ~12ms |

> pgvector 0.7+ HNSW 效能已大幅提升，在 500K 以下場景完全足夠。

### 成本隨規模變化

| 規模 | 基礎設施/月 | 向量 DB 成本 | 每租戶分攤 | API/租戶 |
|------|-----------|------------|----------|---------|
| 1 家 | ~$93 | $0（含在 Cloud SQL） | $93 | ~$0.5-20 |
| 10 家 | ~$120 | $0（pgvector 足夠） | $12 | ~$0.5-20 |
| 30 家 | ~$150 | $0（pgvector 調優） | $5 | ~$0.5-20 |
| 100 家 | ~$200 | +$70（Qdrant 自架 GCE） | $2.7 | ~$0.5-20 |
| 1000 家 | ~$500 | +$300（Qdrant 3-node cluster） | $0.8 | ~$0.5-20 |

### 轉折點決策矩陣

| 決策因素 | pgvector 適合 | Qdrant 適合 |
|---------|-------------|------------|
| 向量數量 | < 5M | > 5M |
| P99 延遲要求 | < 50ms 可接受 | 需要 < 15ms |
| 維運能力 | GCP 託管省事 | 有人力管理 infra |
| 混合查詢需求 | SQL + vector 混合（原生 SQL 強項） | 純向量搜尋 |
| 成本 | 零額外（含在 Cloud SQL） | 需額外 $70-300/月 |

> **核心訊息**：pgvector → Qdrant 遷移不需改業務邏輯。DDD 架構優勢：只換 Infrastructure 層的 Repository 實作，Domain 和 Application 層零修改。

## 四、定價策略

### 方案 A：成本加成（Cost-Plus Markup）

以實際成本加上固定 markup（通常 2-3 倍），適合初期定價：

| 規模 | 實際成本/租戶 | 2x Markup | 3x Markup |
|------|-------------|----------|----------|
| 3 家 | ~$44（1,417 TWD） | 2,834 TWD | 4,251 TWD |
| 10 家 | ~$26（837 TWD） | 1,674 TWD | 2,511 TWD |

### 方案 B：價值定價 — 傳統方案的 40-50%（建議）

以市場上傳統方案的定價為錨點，收其 40-50%：

> 傳統方案參考定價：18M tokens / 50,000 TWD。換算每租戶 1,000 次對話約 5,000-16,667 TWD/月（依使用率）。

| 使用率 | 租戶數 | 傳統方案分攤 | 自建半價定價 | 我方成本 | 毛利率 |
|-------|--------|------------|-----------|---------|--------|

| 5% | 6 家 | 8,333 TWD | **4,167 TWD** | 1,964 TWD | **53%** |
| 3% | 10 家 | 5,000 TWD | **2,500 TWD** | 1,179 TWD | **53%** |

### 方案 C：分級 SaaS 訂閱

## 六、LLM 降價趨勢

### 歷史降幅

| 時間 | 事件 | 影響 |
|------|------|------|
| 2024 → 2025 | GPT-4o 取代 GPT-4 | 同等品質，價格降 75% |
| 2025 → 2026 | GPT-5 系列 + DeepSeek V3 | 整體市場價格再降 50-80% |
| 持續 | 開源模型崛起（Llama、Mistral） | 長期價格壓力持續 |

### 未來成本預估

| 時間 | 預期 LLM 價格變化 | 自建每次對話成本 |
|------|-----------------|---------------|
| 2026 現在 | 基準 | 2.3-2.6 TWD |
| 2026 下半年 | 預估再降 30-50% | 1.2-1.8 TWD |
| 2027 | 預估再降 50% | < 1 TWD |

> 當每次對話成本低於 1 TWD 時，AI 客服的邊際成本趨近於零。

### 自建 vs 傳統方案的降價受益差異

- **自建系統**：LLM 降價 → 營運成本自動降低 → 毛利率提升或可讓利給客戶
- **傳統方案**：平台方吃了中間利潤，降價紅利不一定會傳給客戶

## 七、成本優化路徑

| 優化手段 | 節省幅度 | 時機 | 說明 |
|---------|---------|------|------|
| **Prompt Caching** | -50~90% | 立即可做 | 重複 system prompt 快取，OpenAI/Anthropic/Google 皆支援 |
| **小模型分流** | -80~95% | 立即可做 | 簡單任務用 nano/Flash-Lite，推理用 mini/Flash |
| **Batch API** | -50% | 非即時任務 | 報表、摘要、批次分析等場景 |
| **Committed Use Discount** | -20~30% | 規模化後 | Cloud SQL 1 年承諾折扣 |
| **多租戶共享基礎設施** | -50~70% per tenant | 客戶增加時 | 單一 Cloud SQL 支撐多租戶 |
| **Cloud Run min: 0** | -30~50% | 低用量時段 | 自動縮零，idle 不計費 |


> **定價基準日**：2026-03-15
> **匯率**：1 USD = 32.2 TWD
> **資料來源**：GCP 官方定價、OpenAI / Anthropic / Google 官方 API 定價
