# NotebookLM 簡報生成 Prompt

請根據上傳的 5 份資料（01~05），生成一份產品簡報，介紹 AI Agent 客服平台。

## 簡報格式規範

- **背景圖片**：使用來源中的圖片作為投影片的背景
- **視覺化**：將來源內容轉化成視覺化簡報，不要只放文字
- **字體**：統一使用「微軟正黑體」
- **配色**：文字與圖片配色必須與背景一致、協調
- **字體規格**：依照上傳的「內文規格」圖片中的設定
- **報告人**：AI 應用 Team
- **日期**：2026/03/16

## 目標受眾

公司管理層（部長級），非技術背景，關心商業價值、成本數字、和市場競爭力。假設受眾了解基本的 AI 客服概念，但不熟悉 RAG、Agent、DDD 等技術細節。

## 簡報風格

- **產品行銷 + 主管簡報導向**：目的是讓管理層理解產品定位、商業價值、和成本效益
- 語氣專業但易懂，用圖表和數字取代大段文字
- 每個概念都要有對比表、數據、或流程圖佐證
- 適合 30-40 分鐘的產品報告 session

## 每頁要求

- **標題**：簡潔有力，一句話點明核心
- **要點**：3-5 個 bullet points，每個 point 要具體、有數據
- **視覺化元素**：每頁至少一個圖表（對比表、流程圖、數據卡片等）
- **講者備註**：每頁附上 2-3 句口語化講解要點

## 特別注意

- 受眾是管理層，所有技術概念必須附上「白話版」說明
- 數字是重點：成本、價差倍數、毛利率、測試數量 — 這些數字要突出顯示
- ReAct 推理迴圈是核心差異化，要讓受眾直覺理解「推理 → 行動 → 觀察」
- 傳統方案對比不指名，一律稱「傳統方案」或「業界常見方案」
- 定價策略的「半價仍 53% 毛利」是最有說服力的數字，要在投影片 11 突出呈現
- 不要遺漏 MCP 作為業界標準的論述（Anthropic 提出，OpenAI/Google/Microsoft 採納）
