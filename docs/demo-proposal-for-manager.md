# RAG AI 客服機器人 — Demo 計畫與預算評估

## 一、需要主管協助提供的資料

為了精確估算 Demo 成本與制定展示策略，需要以下資訊：

### 1. 現有系統月流量

請提供目前客服系統的 **每家客戶每月網站流量**（例如：50,000 次/月）。
我會以此流量為基礎，提供兩種成本估算：

- **保守估算（100% 轉換率）**：假設所有訪客都使用 AI 客服，計算成本上限
- **實際估算（2-10% 轉換率）**：根據業界經驗調降至合理比例

以 100% 高估方式仍具價格優勢時，即可直接證明我們方案的成本競爭力。

### 2. LLM 模型選擇（三選一）

以下為 2026 年 3 月市場上 CP 值最高的三個 AI 模型，請選擇一個作為 Demo 主力：

| 排名 | 模型 | 廠商 | 每次對話成本 | 3,000 次/月費用 | 特點 |
|------|------|------|-------------|----------------|------|
| 1 | **Claude Sonnet 4.6** | Anthropic | $0.054 | $162/月 | 效果最好，推理與指令遵循能力最強 |
| 2 | **GPT-5** | OpenAI | $0.030 | $90/月 | 最新旗艦模型，性價比優，工具生態系最完整 |
| 3 | **Gemini 2.5 Pro** | Google | $0.030 | $90/月 | 與 GCP 帳單整合，免額外開帳號 |

> 三者效果差距不大，主要差異在價格與整合便利性。
> 若無特別偏好，建議選擇 **GPT-5**（性價比最優 + 生態系最成熟）。

## 二、Demo 執行計畫

確認上述資訊後，將依以下步驟進行：

### Step 1：模型鎖定

根據主管選定的模型進行後續開發與測試。

### Step 2：Token 消耗實測

使用選定模型，以實際 Demo 情境跑 **5 輪完整對話**，
記錄每輪的 Input / Output / Embedding Token 消耗量，
作為正式成本估算的基礎數據。

### Step 3：成本估算

以現有系統月流量為基礎，提供兩種估算方式：

- **保守估算（100% 轉換率）**：假設每位訪客都觸發 5 輪完整對話，計算成本上限
- **實際估算（調整轉換率）**：根據實測數據調降至合理比例（通常 2-10%）

兩種估算都會提供，方便與競品方案做對比。
以 100% 轉換率估算仍具價格優勢時，即可證明我們方案的成本競爭力。

### Step 4：Demo 對話展示

使用同一組提示詞設定，分別模擬 **兩家不同產業客戶** 的客服對話場景，
展示 AI 客服的實際回答品質與適應性。

### Step 5：完整報價

最終報價 = LLM 成本 + 基礎設施成本

| 項目 | 說明 | 估算 |
|------|------|------|
| LLM 成本 | 依模型 × 流量 × 實測 token 計算 | 待實測 |
| 基礎設施 | GCP 雲端服務（資料庫、伺服器、向量搜尋等） | ~$70/月起 |
| Infra 緩衝 | 正式上線需經 Infra 單位核價，保守以 +20% 高估我方成本，確保與競品比價時仍具優勢 | +20% |
| **預估總成本** | | **待實測後提供** |

## 三、我們產品的核心優勢

在 Demo 時可針對不同對象強調以下差異化優勢：

1. **功能可擴充性高**
   - 模組化架構設計，可依客戶需求快速新增功能
   - 支援多租戶（Multi-tenant），一套系統服務多家客戶

2. **AI 模型可替換**
   - 不綁定單一 AI 廠商，可隨時切換 OpenAI / Claude / Gemini
   - 未來模型降價或更強時，可無痛升級

3. **知識庫自主管理**
   - 客戶可自行上傳、更新產品文件與 FAQ
   - AI 回答基於客戶自有資料，降低幻覺風險

4. **成本透明可控**
   - 按量計費，用多少算多少
   - 提供完整的成本儀表板，隨時掌握費用
